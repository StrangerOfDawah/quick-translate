# Решения и известные ловушки

Этот документ фиксирует причины, а не только текущее состояние. Меняйте решение
только вместе с тестами, документацией и явным объяснением новой причины.

## D-001. Целевой язык всегда русский

Sensemark автоматически определяет исходный язык, но не предлагает выбор языка
результата. Это сохраняет интерфейс быстрым и уменьшает вариативность prompt.

Следствие: выделение только на русском обрабатывается локально без карточки и
без API.

## D-002. Сначала локальная очистка, затем модель

DOM выделения может содержать скрытый дубль, подпись кнопки, шрифтовой глиф,
перевод сайта или управляющий символ. Передавать это модели дорого и
ненадёжно, поэтому доказанный структурный шум убирается в
`selection-text.js`.

Нельзя удалять текст только потому, что он выглядит необычно. Очистка должна
опираться на DOM-сигнал, известный паттерн сайта или управляющую категорию
Unicode.

## D-003. Несколько языков — структурированный ответ

Многоязычный перевод имеет секции с письменностью и языком, чтобы UI не
смешивал результаты. Русская часть остаётся без изменений.

Исключение: короткий параллельный заголовок на двух языках может быть переведён
один раз. Это предотвращает дублирование и лишний repair-запрос.

## D-004. Не показывать пустую карточку

Карточка на странице создаётся только после появления видимого перевода.
Служебная метка и пустой streaming scaffold не считаются видимым ответом.

Причина: пустое окно загрузки отвлекает и создаёт впечатление ошибки.

## D-005. Repair — не обычная часть перевода

Второй API-запрос расходует токены и увеличивает задержку. Он разрешён только
когда локальная валидация называет конкретную ошибку ответа. Нельзя использовать
repair для определения языка, улучшения стиля или исправления необязательного
форматирования.

## D-006. Неизвестные термины имеют отдельный UI

Имя, название, бренд, никнейм, опечатка или выдуманный термин не должны
выглядеть как обычный перевод. `[[reference]]` включает отдельную янтарную
карточку, категорию и осторожное объяснение. Обычное словарное слово вроде
`Why` всегда предпочитает перевод, даже с заглавной буквы.

## D-007. Toolbar popup остаётся компактным Chrome popup

Ручной перевод реализован обычным HTML/CSS/JS без React. Chrome вычисляет
размер окна по содержимому.

В июле 2026 года попытка сделать корень edge-to-edge через
`height: 100%` для `html/body` и `.shell` растянула popup в большое пустое
окно. Изменение было отменено. Поэтому:

- не переносите правила layout обычной веб-страницы в Chrome popup;
- не задавайте процентную высоту всей корневой цепочке;
- проверяйте собранное расширение через `chrome://extensions`;
- сохраняйте текущую компактную внешнюю геометрию, пока пользователь явно не
  утвердит проверенный вариант.

## D-008. Масштаб ограничен по скорости

Трекпад создаёт много wheel-событий. Масштаб меняется небольшими шагами с rate
limit и пределами, иначе текст «улетает» за одно движение.

## D-009. Тест обязателен для каждой фичи и каждого бага

Новый пользовательский сценарий должен иметь автоматический happy path и
релевантную границу. Исправление бага должно оставлять регрессионный тест с
исходным примером. Coverage — дополнительная страховка, но не замена
содержательному тесту.

Если изменение затрагивает количество API-запросов, тест должен проверять
счётчик запросов. Если затрагивает UI — тест должен проверять видимость и
состояние DOM. Если затрагивает отмену — тест должен подтвердить disconnect или
abort.

## D-010. Сначала платформа и проверенные решения

Перед новой библиотекой проверяются стандартные Web API, Chrome APIs и
существующие зрелые решения. Своя реализация допустима, когда задача мала,
специфична для Sensemark и покрыта тестами. Runtime-зависимость не добавляется
без оценки размера, лицензии, безопасности и расхода ресурсов.
